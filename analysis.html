Norman vs. GOMS Comparison

Comparison of Norman’s action cycle stages and GOMS concepts using a Wi-Fi toggle task
Norman’s Stage Real-World Example GOMS Interpretation Alignment Analysis
Goal “I want Wi-Fi to be turned on.” Goal: Enable Wi-Fi.
This is the top-level objective in GOMS. 		Direct correspondence between Norman’s goal and GOMS goal.
Plan User decides to use the quick toggle rather than opening full settings. Method: Use quick-toggle method.
Selection rules apply only if multiple methods are modeled. 		Norman’s planning stage maps conceptually to GOMS methods.
Specify User visually searches for and identifies the Wi-Fi icon. Not explicitly modeled in GOMS.
Assumed knowledge of correct object. 		Key divergence: Norman models perception; GOMS abstracts it away.
Execute User taps the Wi-Fi toggle on the screen. Operator: Tap or click.
Modeled in KLM as a primitive action. 		Both models account for physical execution.
Evaluate User observes feedback (icon color change, status bar indicator). Not modeled in GOMS.
Assumes correct and immediate feedback. 		Fundamental difference: Norman closes the Gulf of Evaluation.
